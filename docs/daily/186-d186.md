# Dia 186 — GraphQL: DataLoader e N+1

## Conceito do Dia

**DataLoader** resolve problema N+1 em GraphQL batcheando queries.

---

## Problema N+1

```graphql
{
  tarefas {
    id
    nome
    projeto {  # N queries!
      nome
    }
  }
}
```

## Sem DataLoader (N+1)

```java
@QueryMapping
public List<Tarefa> tarefas() {
    List<Tarefa> tarefas = service.listar();
    tarefas.forEach(t -> {
        Projeto p = projetoService.buscarPorId(t.getProjetoId()); // 1 query por tarefa!
    });
    return tarefas;
}
```

## Com DataLoader

```java
@Component
public class TarefaDataLoader {
    
    @Autowired
    private GraphQLDataLoaderRegistry registry;
    
    @Autowired
    private ProjetoRepository projetoRepo;
    
    @PostConstruct
    public void registerLoaders() {
        registry.forType(Projeto.class)
            .withBatchLoader(new BatchLoader<Long, Projeto>() {
                @Override
                public CompletionStage<List<Projeto>> load(List<Long> ids) {
                    return CompletableFuture.supplyAsync(() -> 
                        projetoRepo.findAllById(ids)
                    );
                }
            });
    }
}
```

## Uso no Resolver

```java
@Service
public class TarefaService {
    
    public CompletableFuture<Projeto> buscarProjeto(Tarefa tarefa) {
        return DataLoaderRegistry.get()
            .getDataLoader(Projeto.class)
            .load(tarefa.getProjetoId());
    }
}
```

## GraphQL Execution

```
1. Query chega: tarefas { projeto { nome } }
2. DataLoader.collect() reúne todos os projeto IDs
3. BatchLoader.load([1,2,3]) → uma query SQL
4. Resultados distribuídos para cada resolver
5. Retorna resposta
```
