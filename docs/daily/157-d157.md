# Dia 157 — Performance: JVM Tuning

## Conceito do Dia

Otimizar a **JVM** para melhor performance: heap size, garbage collector, e métricas.

---

## Onde Estudar

- [Guia: JVM Performance](https://docs.oracle.com/javase/8/docs/technotes/guides/vm/)

---

## Flags de JVM

```bash
java -Xms512m -Xmx2g \
     -XX:+UseG1GC \
     -XX:MaxGCPauseMillis=200 \
     -XX:+HeapDumpOnOutOfMemoryError \
     -XX:HeapDumpPath=/var/logs \
     -Djava.security.egd=file:/dev/./urandom \
     -jar app.jar
```

| Flag | Significado |
|------|-------------|
| -Xms | Heap inicial |
| -Xmx | Heap máximo |
| -Xss | Stack size por thread |

## Garbage Collectors

| GC | Uso |
|----|-----|
| G1GC | Padrão moderno, balanceado |
| ZGC | Baixa latência, heaps grandes |
| Shenandoah | Similar a ZGC |

## application.yml (Spring)

```yaml
spring:
  jvm:
    target: memory

management:
  metrics:
    export:
      prometheus:
        enabled: true
  endpoints:
    web:
      exposure:
        include: health,prometheus,metrics
```

## Métricas Importantes

```bash
# Heap usage
jstat -gc <pid>

# Thread dump
jstack <pid>

# Heap dump
jmap -dump:file=heap.hprof <pid>
```
