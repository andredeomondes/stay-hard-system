# Dia 135 — Observabilidade: Tracing Distribuído

## Conceito do Dia

Em microservices, tracing conecta requisições entre serviços. **OpenTelemetry** + **Jaeger** para observabilidade.

## Configuração OpenTelemetry

```xml
<dependency>
    <groupId>io.opentelemetry</groupId>
    <artifactId>opentelemetry-api</artifactId>
</dependency>
<dependency>
    <groupId>io.opentelemetry.instrumentation</groupId>
    <artifactId>opentelemetry-spring-boot-starter</artifactId>
</dependency>
```

```yaml
spring:
  application:
    name: tarefa-service

otel:
  service:
    name: tarefa-service
  exporter:
    otlp:
      endpoint: http://jaeger:4317

management:
  tracing:
    sampling:
      probability: 1.0
```

## Trace Manual

```java
import io.opentelemetry.api.trace.Tracer;
import io.opentelemetry.api.trace.Span;

@Service
@RequiredArgsConstructor
public class TarefaService {
    
    private final Tracer tracer;
    
    public Tarefa criar(CriarTarefaRequest request) {
        Span span = tracer.spanBuilder("criarTarefa").startSpan();
        
        try (scope = span.makeCurrent()) {
            span.setAttribute("tarefa.nome", request.nome());
            
            log.info("Criando tarefa: {}", request.nome());
            
            Tarefa tarefa = repository.save(tarefa);
            
            span.setAttribute("tarefa.id", tarefa.getId());
            
            return tarefa;
        } catch (Exception e) {
            span.recordException(e);
            throw e;
        } finally {
            span.end();
        }
    }
}
```

## Jaeger no Docker Compose

```yaml
jaeger:
  image: jaegertracing/all-in-one:1.52
  ports:
    - "16686:16686"  # UI
    - "4317:4317"    # OTLP gRPC
    - "4318:4318"    # OTLP HTTP
  environment:
    - COLLECTOR_OTLP_ENABLED=true
```
