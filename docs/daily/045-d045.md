# Dia 45 — Factory Pattern

## Conceito do Dia

**Factory Method** encapsula a criação de objetos. O cliente não sabe qual classe concreta está sendo instanciada.

## Código: ConnectionFactory

```java
// Produto (interface)
interface Conexao {
    void conectar();
    void desconectar();
}

// Implementações concretas
class ConexaoMySQL implements Conexao {
    @Override
    public void conectar() {
        System.out.println("Conectando ao MySQL...");
    }
    @Override
    public void desconectar() {
        System.out.println("Desconectando do MySQL...");
    }
}

class ConexaoPostgreSQL implements Conexao {
    @Override
    public void conectar() {
        System.out.println("Conectando ao PostgreSQL...");
    }
    @Override
    public void desconectar() {
        System.out.println("Desconectando do PostgreSQL...");
    }
}

// Factory
enum TipoBanco { MYSQL, POSTGRESQL }

class ConnectionFactory {
    public static Conexao criar(TipoBanco tipo) {
        return switch (tipo) {
            case MYSQL -> new ConexaoMySQL();
            case POSTGRESQL -> new ConexaoPostgreSQL();
        };
    }
}

// Cliente usa
public class Main {
    public static void main(String[] args) {
        Conexao conn = ConnectionFactory.criar(TipoBanco.MYSQL);
        conn.conectar();
    }
}
```
