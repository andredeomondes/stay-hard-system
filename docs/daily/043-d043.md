# Dia 43 — SOLID: Liskov Substitution Principle

## Conceito do Dia

Objetos de uma superclasse devem poder ser substituídos por objetos de subclasses **sem quebrar** a aplicação.

"Um pato que é também um robotic duck deve conseguir grasnar e nadar." — Nãoforce um Pato a implementar `Voar` se for um Pato-da-Barra.

## Mini-Projeto: Veículos

```java
// VIOLAÇÃO DE LSP
class Veiculo {
    void abrirPorta() {}
}

class Carro extends Veiculo { } // OK
class Moto extends Veiculo {
    @Override
    void abrirPorta() { 
        throw new UnsupportedOperationException("Moto não tem portas!"); 
    }
}

// CORRETO: Composition
interface ComPortas { void abrirPorta(); }
interface ComRodas { void acelerar(); }

class Carro implements ComPortas, ComRodas { }
class Moto implements ComRodas { }
```
