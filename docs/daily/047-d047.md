# Dia 47 — Singleton Pattern

## Conceito do Dia

Garante que uma classe tem **apenas uma instância** e fornece um ponto global de acesso a ela.

⚠️ **Use com moderação** - pode criar acoplamento forte e dificultar testes.

## Código: Singleton com Enum

```java
// FORMA RECOMENDADA
enum DatabaseConnection {
    INSTANCE;
    
    private Connection conn;
    
    DatabaseConnection() {
        // inicialização custosa
        this.conn = criarConexao();
    }
    
    public Connection getConnection() {
        return conn;
    }
    
    private Connection criarConexao() {
        // lógica de conexão
        return null;
    }
}

// Uso
Connection conn = DatabaseConnection.INSTANCE.getConnection();
```

```java
// FORMA CLÁSSICA (se não puder usar enum)
class Configuracao {
    private static volatile Configuracao instancia;
    
    private Configuracao() {}
    
    public static Configuracao getInstancia() {
        if (instancia == null) {
            synchronized (Configuracao.class) {
                if (instancia == null) {
                    instancia = new Configuracao();
                }
            }
        }
        return instancia;
    }
}
```

---

## Quando USAR Singleton

- Conexão com banco de dados (uma só conexão)
- Configurações globais da aplicação
- Logger (um só ponto de escrita)

## Quando NÃO USAR

- Objetos que precisam de estado diferente por request
- Objetos que você quer mockar em testes

---

## Aplicar no Stay Hard

Se necessário, use Singleton para `ConfiguracaoApp` (leitura de configurações).
