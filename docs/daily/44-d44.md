# Dia 44 — SOLID: Interface Segregation + Dependency Inversion

## Conceito do Dia

**ISP**: Interfaces específicas valem mais que interfaces genéricas.
**DIP**: Dependa de abstrações, não de implementações concretas.

## Checklist de Código

- [ ] Identificar interfaces "gordas" com muitos métodos
- [ ] Dividir em interfaces menores e coesas
- [ ] Injetar dependências via construtor (não via `new`)

---

## Mini-Projeto: Printer Interface

```java
// ANTES: Interface gorda
interface Printer {
    void printPdf();
    void printWord();
    void printImage();
}

// DEPOIS: Interfaces segregadas
interface PdfPrinter { void printPdf(); }
interface ImagePrinter { void printImage(); }

// Cliente só usa o que precisa
class RelatorioService {
    private final PdfPrinter printer;
    
    RelatorioService(PdfPrinter printer) {
        this.printer = printer; // DIP na prática
    }
}
```

## Aplicar no Stay Hard

```java
// DIP na prática
class TarefaRepository {
    private final Persistencia persistence; // abstração, não implementação
    
    TarefaRepository(Persistencia persistence) {
        this.persistence = persistence;
    }
}
```
