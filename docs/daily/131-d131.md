# Dia 131 — Microservices: Introdução

## Conceito do Dia

**Microservices** divide uma aplicação em serviços pequenos e independentes. Cada um com seu próprio deploy, banco de dados e responsabilidade.

## Monolith vs Microservices

```
MONOLITH                           MICROSERVICES
+---------+                         +---------+
|  API    |                         | Auth    | → auth-service
|  Web    |                         +---------+
|  Admin  |                         +---------+
|  Jobs   |                         | Tarefas | → tarefas-service
+---------+                         +---------+
|  ORM    |                         +---------+
|  MySQL  |                         | Notifs  | → notificacao-service
+---------+                         +---------+

- 1 deploy                       - Deploys independentes
- 1 banco                        - Bancos separados
- Acoplado                       - Baixo acoplamento
- Simples inicialmente            - Complexo, mas escalável
```

## Domínios do Stay Hard

| Serviço | Responsabilidade |
|---------|-----------------|
| `auth-service` | Login, registro, JWT |
| `tarefa-service` | CRUD de tarefas |
| `projeto-service` | Projetos |
| `notificacao-service` | Emails, push |
| `gateway-service` | API Gateway |
