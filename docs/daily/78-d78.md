# Dia 78 — JPA: Hibernate vs EclipseLink

## Conceito do Dia

**Hibernate** é a implementação JPA mais popular. **EclipseLink** é a implementação de referência. O Spring Boot usa Hibernate por padrão.

## Configuração Hibernate

```yaml
spring:
  jpa:
    properties:
      hibernate:
        dialect: org.hibernate.dialect.PostgreSQLDialect
        format_sql: true
        use_sql_comments: true
        jdbc:
          time_zone: UTC
    hibernate:
      ddl-auto: validate  # ou create, update, none
    open-in-view: false  # evitar LazyInitializationException
```

## DDL Auto Modes

| Mode | Comportamento |
|------|---------------|
| none | Não faz nada |
| validate | Só valida, não muda |
| update | Atualiza schema |
| create | Cria tabelas (dropa existentes) |
| create-drop | Cria no start, dropa no stop |

⚠️ **NUNCA use create/create-drop em produção!**

## Dicas de Performance

```yaml
spring:
  jpa:
    properties:
      hibernate:
        default_batch_fetch_size: 25  # N+1 problem
        jdbc:
          batch_size: 50               # Insert batching
        order_inserts: true
        order_updates: true
```
