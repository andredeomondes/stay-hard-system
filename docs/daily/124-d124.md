# Dia 124 — Docker: Docker Network e Volumes

## Conceito do Dia

**Networks** permitem comunicação entre containers. **Volumes** persistem dados entre execuções.

---

## Onde Estudar

- [Docs: Networking](https://docs.docker.com/compose/networking/)
- [Docs: Volumes](https://docs.docker.com/storage/volumes/)

## Networks

```yaml
version: '3.8'

services:
  api:
    networks:
      - frontend
      - backend

  nginx:
    image: nginx:alpine
    networks:
      - frontend
    ports:
      - "80:80"
    depends_on:
      - api

networks:
  frontend:
    driver: bridge
  backend:
    driver: bridge
    internal: true  # Sem acesso ao mundo externo
```

## Volumes

```yaml
version: '3.8'

services:
  api:
    volumes:
      # Volume nomeado
      - app_data:/app/data
      
      # Bind mount (caminho do host)
      - ./logs:/app/logs
      
      # Somente leitura
      - ./config:/app/config:ro

  db:
    volumes:
      # Volume nomeado (recomendado)
      - db_data:/var/lib/postgresql/data

volumes:
  app_data:
  db_data:
```

## Tipos de Volumes

| Tipo | Uso |
|------|-----|
| anonymous | Temporário, não persiste |
| named | Persiste entre recreações (recomendado) |
| bind mount | Sincroniza com host (dev) |
| tmpfs | Na memória, não persiste (segredos) |
