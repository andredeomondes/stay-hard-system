# Dia 42 — SOLID: Open/Closed Principle

## Conceito do Dia

Entidades devem ser **abertas para extensão** e **fechadas para modificação**. Adicione comportamento novo sem alterar código existente.

## Mini-Projeto: Formatos de Exportação

```java
// ANTES: Switch terrível
class Exportador {
    String exportar(Tarefa tarefa, String formato) {
        if (formato.equals("json")) return toJson(tarefa);
        if (formato.equals("csv")) return toCsv(tarefa);
        // toda vez que muda, mexe aqui
    }
}

// DEPOIS: extensível
interface ExportadorTarefa {
    String exportar(Tarefa tarefa);
}

class ExportadorJson implements ExportadorTarefa { ... }
class ExportadorCsv implements ExportadorTarefa { ... }
```
