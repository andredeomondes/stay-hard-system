# Dia 177 — System Design: URL Shortener

## Conceito do Dia

Projetar um encurtador de URLs (como bit.ly).

---

## Requirements

```
Functionais:
- Criar short URL a partir de long URL
- Redirecionar short → long
- Analytics (cliques)

Não-funcionais:
- 100M URLs/dia
- 10B redirects/dia
- Latência < 10ms
```

## API Design

```
POST   /urls
  Body: {"longUrl": "https://..."}
  Response: {"shortUrl": "abc123"}

GET    /{shortUrl}
  Response: 302 → longUrl

GET    /urls/{shortUrl}/analytics
  Response: {"clicks": 1234, "lastClick": "..."}
```

## Storage

```
id: 1234567890 (base62: "14lR9u")
long_url: https://exemplo.com/path
created_at: 2026-03-29
clicks: 0
```

### Database: PostgreSQL

```sql
CREATE TABLE urls (
    id BIGSERIAL PRIMARY KEY,
    short_code VARCHAR(10) UNIQUE NOT NULL,
    long_url TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT NOW(),
    clicks INTEGER DEFAULT 0,
    INDEX idx_short_code (short_code)
);
```

### Encoding

```java
// Base62 encoding (a-z, A-Z, 0-9)
private static final String CHARS = 
    "abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789";

public String encode(long id) {
    StringBuilder sb = new StringBuilder();
    while (id > 0) {
        sb.append(CHARS.charAt((int)(id % 62)));
        id /= 62;
    }
    return sb.reverse().toString();
}
```

### Cache

```java
@Cacheable(value = "urls", key = "#shortCode")
public Optional<UrlEntity> findByShortCode(String shortCode) {
    return repository.findByShortCode(shortCode);
}
```

## Caching Strategy

```
1. Check Redis for short_code
2. If miss, query DB
3. Store in Redis with TTL (24h)
4. Increment click counter async
```
