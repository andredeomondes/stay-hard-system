# Dia 161 — Testing: TDD na Prática

## Conceito do Dia

**TDD (Test-Driven Development)**: primeiro o teste, depois o código. Vermelho → Verde → Refatorar.

---

## Ciclo TDD

```
1. RED: Escreva teste que falha
2. GREEN: Escreva código mínimo para passar
3. REFACTOR: Melhore o código mantendo testes passando
```

## Exemplo TDD

```java
// 1. RED - Escreva o teste primeiro
class TarefaServiceTest {
    
    @Test
    void criar_comNomeNulo_deveLancarExcecao() {
        TarefaService service = new TarefaService(null);
        
        assertThrows(TarefaInvalidaException.class, () -> {
            service.criar(null);
        });
    }
    
    @Test
    void criar_comNomeValido_deveRetornarTarefa() {
        // Arrange
        TarefaRepository repo = mock(TarefaRepository.class);
        when(repo.save(any())).thenAnswer(inv -> {
            Tarefa t = inv.getArgument(0);
            t.setId(1L);
            return t;
        });
        
        TarefaService service = new TarefaService(repo);
        
        // Act
        Tarefa resultado = service.criar("Nova tarefa");
        
        // Assert
        assertNotNull(resultado);
        assertEquals("Nova tarefa", resultado.getNome());
        assertFalse(resultado.isConcluida());
    }
}

// 2. GREEN - Implemente o mínimo
class TarefaService {
    
    Tarefa criar(String nome) {
        if (nome == null || nome.isBlank()) {
            throw new TarefaInvalidaException("Nome é obrigatório");
        }
        return repository.save(new Tarefa(nome));
    }
}
```
