# Dia 64 — Spring Boot: application.properties e YAML

## Conceito do Dia

Configurações da aplicação ficam em `application.properties` ou `application.yml`. O Spring Boot configura automaticamente baseado nessas propriedades.

---

## Onde Estudar

## application.properties

```properties
# Server
server.port=8080
server.servlet.context-path=/api

# Application
spring.application.name=stay-hard-api

# Dev (não usar em produção!)
spring.jpa.show-sql=true
spring.jpa.hibernate.ddl-auto=update

# Logging
logging.level.br.com.stayhard=DEBUG
logging.level.org.springframework.web=INFO
```

## application.yml (mais limpo)

```yaml
server:
  port: 8080
  servlet:
    context-path: /api

spring:
  application:
    name: stay-hard-api
  datasource:
    url: jdbc:postgresql://localhost:5432/stayhard
    username: postgres
    password: ${DB_PASSWORD}
  jpa:
    hibernate:
      ddl-auto: update
    show-sql: true

logging:
  level:
    br.com.stayhard: DEBUG
    org.hibernate.SQL: DEBUG
```

## Profiles

```yaml
# application-dev.yml
spring:
  datasource:
    url: jdbc:h2:mem:testdb

---
# application-prod.yml
spring:
  datasource:
    url: jdbc:postgresql://prod-server:5432/stayhard
  jpa:
    hibernate:
      ddl-auto: validate
```

Rodar com profile:
```bash
java -jar app.jar --spring.profiles.active=prod
```
