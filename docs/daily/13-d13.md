## 🎯 O QUE VOCÊ VAI APRENDER HOJE

### Conceito:
**Maven Lifecycle — Ciclo de vida do build**

### Definição em 1 frase:
Maven tem fases padronizadas: validate → compile → test → package → install → deploy.

### Fluxo:
```
validate    → Verifica projeto
    ↓
compile     → Compila src/main/java
    ↓
test        → Roda testes src/test/java
    ↓
package     → Gera JAR
    ↓
install    → Instala no repositório local
    ↓
deploy     → Publica em servidor remoto
```

## 💻 CHECKLIST DE CÓDIGO

### 📁 Exercício 1: Limpar

```bash
# clean: Remove pasta target/
mvn clean

# Verifica: target/ foi deletada
ls -la
```

**Antes:**
```
projeto/
├── target/         ← compilado
├── src/
└── pom.xml
```

**Depois:**
```
projeto/
├── (target deletada)
├── src/
└── pom.xml
```

---

### 📁 Exercício 2: Compile

```bash
# compile: Compila src/main/java → target/classes/
mvn compile

# Verifica:
ls target/classes/com/estudo/
# App.class
```

---

### 📁 Exercício 3: Test

```bash
# test: Compila + Roda testes
mvn test

# Verifica em target/surefire-reports/
cat target/surefire-reports/*.txt
```

**Output esperado:**
```
Tests run: 1, Failures: 0, Errors: 0, Skipped: 0
```

---

### 📁 Exercício 4: Package

```bash
# package: Compila + Testa + Gera JAR
mvn package

# Verifica:
ls target/*.jar
# meu-projeto-1.0.0.jar
```

---

### 📁 Exercício 5: Combinar Comandos

```bash
# clean + compile
mvn clean compile

# clean + test
mvn clean test

# clean + package
mvn clean package

# clean + install (mais completo)
mvn clean install

# clean + compile + test (pular package)
mvn clean test
```

---

### 📁 Exercício 6: Skip Tests

```bash
# Pular testes (mais rápido)
mvn clean package -DskipTests

# Pular testes e compilation de testes
mvn clean package -Dmaven.test.skip=true
```

**⚠️ Cuidado:** Pular testes não é recomendado em produção!

---

### 📁 Exercício 7: Rodar Aplicação

```bash
# Opção 1: Via Java direto
java -cp "target/classes:target/dependency/*" com.estudo.App

# Opção 2: Com Maven
mvn exec:java -Dexec.mainClass="com.estudo.App"

# Opção 3: JAR executável (com Spring Boot ou shade plugin)
```

