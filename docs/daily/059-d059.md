# Dia 59 — Exception Handling em Web

## Conceito do Dia

Erros HTTP devem ser tratados adequadamente. Use `web.xml` ou anotações para mapear exceções para páginas de erro.

## web.xml

```xml
<error-page>
    <error-code>404</error-code>
    <location>/WEB-INF/views/erro-404.jsp</location>
</error-page>

<error-page>
    <error-code>500</error-code>
    <location>/WEB-INF/views/erro-500.jsp</location>
</error-page>

<error-page>
    <exception-type>br.com.stayhard.exception.TarefaNaoEncontradaException</exception-type>
    <location>/WEB-INF/views/erro-404.jsp</location>
</error-page>
```

## Páginas de erro

```jsp
<%@ page isErrorPage="true" contentType="text/html" %>
<!DOCTYPE html>
<html>
<head>
    <title>Erro</title>
</head>
<body>
    <h1>Oops! Algo deu errado</h1>
    <p>Código: <%= request.getAttribute("javax.servlet.error.status_code") %></p>
    <p>Mensagem: <%= exception != null ? exception.getMessage() : "Desconhecida" %></p>
    <a href="/">Voltar para home</a>
</body>
</html>
```

## ServletExceptionHandler

```java
@WebServlet("/error-handler")
public class ServletExceptionHandler extends HttpServlet {
    
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) 
            throws ServletException, IOException {
        
        Integer status = (Integer) req.getAttribute("jakarta.servlet.error.status_code");
        String msg = (String) req.getAttribute("jakarta.servlet.error.message");
        
        resp.setContentType("application/json");
        resp.setStatus(status != null ? status : 500);
        
        resp.getWriter().write(String.format("""
            {
                "erro": %d,
                "mensagem": "%s"
            }
            """, status, msg));
    }
}
```
