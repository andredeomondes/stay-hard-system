## 💻 CHECKLIST DE CÓDIGO

### 📁 Exercício 1: API Testing com curl

**Objetivo:** Testar APIs públicas usando curl

**1. GET — Buscar usuário:**
```bash
# Listar todos usuários
curl https://jsonplaceholder.typicode.com/users

# Buscar usuário específico
curl https://jsonplaceholder.typicode.com/users/1

# Filtrar com query params
curl "https://jsonplaceholder.typicode.com/users?username=Bret"
```

**2. POST — Criar usuário:**
```bash
curl -X POST https://jsonplaceholder.typicode.com/users \
  -H "Content-Type: application/json" \
  -d '{
    "name": "André de Omondes",
    "username": "andredev",
    "email": "andre@email.com"
  }'
```

**3. PUT — Atualizar:**
```bash
curl -X PUT https://jsonplaceholder.typicode.com/users/1 \
  -H "Content-Type: application/json" \
  -d '{
    "id": 1,
    "name": "Nome Atualizado",
    "username": "novoUsername",
    "email": "novo@email.com"
  }'
```

**4. DELETE — Remover:**
```bash
curl -X DELETE https://jsonplaceholder.typicode.com/users/1
```

**Checklist:**
- [ ] GET funciona (status 200)
- [ ] POST funciona (status 201)
- [ ] PUT funciona (status 200)
- [ ] DELETE funciona (status 200 ou 204)

### 📁 Exercício 2: curl com Headers

**Objetivo:** Enviar headers personalizados

```bash
# Enviar header de autenticação
curl -H "Authorization: Bearer meu-token-123" \
     https://api.exemplo.com/dados

# Enviar Content-Type
curl -H "Content-Type: application/json" \
     -d '{"nome": "André"}' \
     https://api.exemplo.com/users

# Ver todos headers da resposta
curl -i https://jsonplaceholder.typicode.com/users/1

# Ver headers apenas (HEAD)
curl -I https://google.com
```

**Checklist:**
- [ ] Authorization header
- [ ] Content-Type header
- [ ] Ver resposta completa (-v)

### 📁 Exercício 3: curl Verbose e Info

```bash
# Verbose (mostra todo processo)
curl -v https://jsonplaceholder.typicode.com/users/1

# Ver output:
# > GET /users/1 HTTP/1.1          ← Request Line
# > Host: jsonplaceholder...        ← Headers
# < HTTP/1.1 200 OK                ← Status
# < Content-Type: application/json ← Headers
# <
# {"id": 1, "name": "...", ...}   ← Body

# Salvar resposta em arquivo
curl -o resposta.json https://jsonplaceholder.typicode.com/users/1

# Timeout
curl --max-time 10 https://api.exemplo.com
```

**Checklist:**
- [ ] Verbose (-v)
- [ ] Salvar em arquivo (-o)
- [ ] Timeout (--max-time)

### 📁 Exercício 4: Autenticação com curl

```bash
# Basic Auth
curl -u usuario:senha https://api.exemplo.com/perfil

# Bearer Token
curl -H "Authorization: Bearer eyJhbGciOiJIUz..." \
     https://api.exemplo.com/dados-privados

# API Key
curl -H "X-API-Key: sua-chave-aqui" \
     https://api.exemplo.com/dados
```

**Checklist:**
- [ ] Basic Auth (-u)
- [ ] Bearer Token (-H)
- [ ] API Key (-H)

## 🔧 COMANDOS DO DIA

```bash
# GET
curl https://api.exemplo.com/dados

# POST
curl -X POST https://api.exemplo.com/dados \
  -H "Content-Type: application/json" \
  -d '{"campo": "valor"}'

# PUT
curl -X PUT https://api.exemplo.com/dados/1 \
  -H "Content-Type: application/json" \
  -d '{"campo": "novo"}'

# DELETE
curl -X DELETE https://api.exemplo.com/dados/1

# Headers
curl -H "Header: valor" https://api.exemplo.com

# Verbose
curl -v https://api.exemplo.com

# Salvar
curl -o arquivo.json https://api.exemplo.com
```
