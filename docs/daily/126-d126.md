# Dia 126 — CI/CD: Deploy com GitHub Actions

## Conceito do Dia

Automatizar deploy para ambientes (staging/prod) usando GitHub Actions e Docker.

---

## Onde Estudar

- [Artigo: Deploy Automation](https://docs.github.com/actions/deployment)

## Secrets no GitHub

```
Settings → Secrets and variables → Actions
```

Adicionar:
- `DOCKER_USERNAME`
- `DOCKER_PASSWORD`
- `SERVER_HOST`
- `SERVER_SSH_KEY`

## .github/workflows/deploy.yml

```yaml
name: Deploy

on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  build-and-push:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Login to Docker Hub
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKER_USERNAME }}
          password: ${{ secrets.DOCKER_PASSWORD }}
      
      - name: Build and push API
        uses: docker/build-push-action@v5
        with:
          context: ./stay-hard-api
          push: true
          tags: ${{ secrets.DOCKER_USERNAME }}/stayhard-api:latest
          cache-from: type=gha
          cache-to: type=gha,mode=max
      
      - name: Build and push Frontend
        uses: docker/build-push-action@v5
        with:
          context: ./stay-hard-frontend
          push: true
          tags: ${{ secrets.DOCKER_USERNAME }}/stayhard-frontend:latest

  deploy:
    needs: build-and-push
    runs-on: ubuntu-latest
    
    steps:
      - name: Deploy to server
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: deploy
          key: ${{ secrets.SERVER_SSH_KEY }}
          script: |
            cd /opt/stayhard
            docker-compose pull
            docker-compose up -d
            docker-compose exec -T api java -jar app.jar
```

## Docker Registry alternativo (ghcr.io)

```yaml
- name: Login to GitHub Container Registry
  uses: docker/login-action@v3
  with:
    registry: ghcr.io
    username: ${{ github.actor }}
    password: ${{ secrets.GITHUB_TOKEN }}
```
