## Usuários em memória

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Bean
    UserDetailsService userDetailsService() {
        UserDetails user = User.builder()
            .username("admin")
            .password(passwordEncoder().encode("admin123"))
            .roles("ADMIN", "USER")
            .build();
        
        UserDetails dev = User.builder()
            .username("dev")
            .password(passwordEncoder().encode("dev123"))
            .roles("USER")
            .build();
        
        return new InMemoryUserDetailsManager(user, dev);
    }
    
    @Bean
    PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```

## Roles vs Authorities

```java
// Roles (conveniência) - prefixo ROLE_ automaticamente
.roles("ADMIN", "USER")
// → authorities: ROLE_ADMIN, ROLE_USER

// Authorities (mais granular)
.authorities("READ", "WRITE", "DELETE")
```

## Testar

```bash
# Sem auth
curl http://localhost:8080/api/tarefas
# 401 Unauthorized

# Com auth básica
curl -u admin:admin123 http://localhost:8080/api/tarefas
# 200 OK
```
