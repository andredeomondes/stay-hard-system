# Dia 63 — Spring Boot: Injeção de Dependência (DI)

## Conceito do Dia

**Inversão de Controle (IoC)**: o Spring gerencia as dependências, não você.
**Injeção de Dependência (DI)**: o Spring "injeta" as dependências nos construtores.

## Checklist de Código

- [ ] Usar `@Service` para classes de serviço
- [ ] Usar `@Repository` para repositórios
- [ ] Usar `@RequiredArgsConstructor` do Lombok (injeção via construtor)

---

## Sem Spring (manual)

```java
public class TarefaController {
    private TarefaService service;
    
    public TarefaController() {
        this.service = new TarefaService(); // acoplamento forte!
    }
}
```

## Com Spring (DI)

```java
@Service
public class TarefaService {
    private final TarefaRepository repository;
    
    public TarefaService(TarefaRepository repository) {
        this.repository = repository;
    }
}

@Repository
public class TarefaRepositoryImpl implements TarefaRepository {
    // implementação
}

@RestController
@RequiredArgsConstructor
public class TarefaController {
    private final TarefaService service;
    // Spring injeta automaticamente!
}
```

## Por que @RequiredArgsConstructor?

```java
// Lombok gera:
public TarefaService(TarefaRepository repository) {
    this.repository = repository;
}
// Sem precisar escrever!
```
