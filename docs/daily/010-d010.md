## 🎯 O QUE VOCÊ VAI APRENDER HOJE

### Conceito:
**RESTful API — Boas práticas para APIs profissionais**

### Definição em 1 frase:
RESTful é um estilo de construir APIs onde tudo é recurso, URLs são caminhos, e HTTP é o protocolo.

### 6 Regras RESTful:

```
1. ✅ Recursos como substantivos: /users, /habits
2. ✅ HTTP Methods como verbos: GET, POST, PUT, DELETE
3. ✅ Status codes corretos: 200, 201, 404, etc.
4. ✅ Sem estado (stateless): cada request é independente
5. ✅ URLs consistentes: /users/{id}/habits
6. ✅ Versionamento: /api/v1/users
```

## 💻 CHECKLIST DE CÓDIGO

### 📁 Exercício 1: API RESTful vs Não-RESTful

**❌ NÃO RESTful (ruim):**
```bash
# Verbos no URL
curl -X POST https://api.com/createUser
curl -X POST https://api.com/deleteUser
curl -X POST https://api.com/updateUser

# Ações como recursos
curl https://api.com/getAllUsers
curl https://api.com/activateUser/1
curl https://api.com/deactivateUser/1
```

**✅ RESTful (bom):**
```bash
# CRUD como recursos
curl https://api.com/users          # GET listar
curl -X POST https://api.com/users   # POST criar
curl https://api.com/users/1        # GET buscar
curl -X PUT https://api.com/users/1 # PUT atualizar
curl -X DELETE https://api.com/users/1 # DELETE remover

# Ações como sub-recursos
curl -X POST https://api.com/users/1/activate   # POST ativar
curl -X POST https://api.com/users/1/deactivate # POST desativar
```

**Checklist:**
- [ ] Identificar URLs ruins
- [ ] Regras de URLs boas
- [ ] Entender: GET=listar, POST=criar, PUT=substituir, DELETE=remover

### 📁 Exercício 2: Nested Resources (Recursos Aninhados)

```bash
# Usuários têm hábitos
# GET /users/1/habits → Lista hábitos do usuário 1
curl https://api.com/users/1/habits

# POST /users/1/habits → Cria hábito para usuário 1
curl -X POST https://api.com/users/1/habits \
  -H "Content-Type: application/json" \
  -d '{"name": "Treinar"}'

# GET /users/1/habits/2 → Hábito 2 do usuário 1
curl https://api.com/users/1/habits/2
```

**Checklist:**
- [ ] URLs aninhadas
- [ ] Entender quando usar aninhamento
- [ ] Entender quando NÃO usar (máximo 2 níveis)

---

### 📁 Exercício 3: Query Params para Filtros

```bash
# Filtrar
GET /habits?priority=HIGH
GET /habits?status=DONE
GET /habits?userId=1&status=TODO

# Ordenar
GET /habits?sort=createdAt&order=desc

# Paginar
GET /habits?page=1&size=10
GET /habits?offset=0&limit=10

# Combinar
GET /habits?status=TODO&priority=HIGH&sort=name&page=1&size=10
```

**Checklist:**
- [ ] Filtros (?status=DONE)
- [ ] Ordenação (?sort=name&order=asc)
- [ ] Paginação (?page=1&size=10)
- [ ] Combinar filtros

---

### 📁 Exercício 4: Versionamento

```bash
# Versão na URL (mais comum)
GET /api/v1/users
GET /api/v2/users

# Header (alternativa)
curl -H "API-Version: 2024-01-01" https://api.com/users
```

**Checklist:**
- [ ] Versionamento na URL
- [ ] Entender quando criar nova versão

### 📁 Exercício 5: Response Format Padrão

```bash
# Sucesso: array para listagem
{
  "data": [
    {"id": 1, "name": "João"},
    {"id": 2, "name": "Maria"}
  ],
  "meta": {
    "page": 1,
    "total": 100,
    "perPage": 10
  }
}

# Sucesso: objeto para singular
{
  "data": {
    "id": 1,
    "name": "João"
  }
}

# Erro: formato consistente
{
  "error": {
    "code": "USER_NOT_FOUND",
    "message": "Usuário não encontrado",
    "details": {}
  }
}
```

**Checklist:**
- [ ] Response wrapper (data)
- [ ] Meta para paginação
- [ ] Error format consistente

## 📊 RESTFUL CHECKLIST

### URL Design
- [ ] Recursos como substantivos (plural)
- [ ] Sem verbos no URL
- [ ] Minúsculas
- [ ] Hífens para separar palavras
- [ ] Máximo 2 níveis de aninhamento

### HTTP Methods
- [ ] GET = listar/buscar
- [ ] POST = criar
- [ ] PUT = substituir
- [ ] PATCH = atualizar parcialmente
- [ ] DELETE = remover

### Status Codes
- [ ] 200 OK para GET/PUT
- [ ] 201 Created para POST
- [ ] 204 No Content para DELETE
- [ ] 400 Bad Request para validação
- [ ] 401 Unauthorized para auth
- [ ] 404 Not Found para não existente
- [ ] 500 Internal Error para bugs

### Headers
- [ ] Content-Type: application/json
- [ ] Accept: application/json
- [ ] Location para 201
