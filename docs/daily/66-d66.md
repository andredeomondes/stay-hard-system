# Dia 66 — Spring Boot: Validação com Bean Validation

## Conceito do Dia

Validação na entrada de dados é essencial. Use **Bean Validation** (`jakarta.validation`) para validar DTOs antes de processar.

---

## Onde Estudar

- [Artigo: Bean Validation](https://www.baeldung.com/jakarta-validations)
- [Docs: Hibernate Validator](https://hibernate.org/validator/documentation/)

---

## Checklist de Código

- [ ] Adicionar dependência validation

---

## Dependência

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-validation</artifactId>
</dependency>
```

## DTO com validação

```java
public record CriarTarefaRequest(
    @NotBlank(message = "Nome é obrigatório")
    @Size(min = 3, max = 100, message = "Nome deve ter 3-100 caracteres")
    String nome,
    
    @NotNull(message = "Categoria é obrigatória")
    Categoria categoria,
    
    @NotNull(message = "Prioridade é obrigatória")
    Prioridade prioridade
) {}
```

## Controller com @Valid

```java
@PostMapping
@ResponseStatus(HttpStatus.CREATED)
public Tarefa criar(@Valid @RequestBody CriarTarefaRequest request,
                    Errors errors) {
    
    if (errors.hasErrors()) {
        throw new ValidationException(errors.getFieldErrors()
            .stream()
            .map(e -> e.getField() + ": " + e.getDefaultMessage())
            .collect(Collectors.joining(", ")));
    }
    
    return service.criar(request);
}
```

## Validações comuns

| Anotação | Uso |
|----------|-----|
| @NotNull | Não pode ser null |
| @NotBlank | String não vazia |
| @NotEmpty | Coleção não vazia |
| @Size | Tamanho de String/Collection |
| @Min, @Max | Valor numérico mínimo/máximo |
| @Email | Formato de email |
| @Pattern | Regex |
| @Past | Data no passado |
| @Future | Data no futuro |
