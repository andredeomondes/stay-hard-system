# Dia 96 — Spring Security: Autorização com Roles

## Conceito do Dia

Controlar quem pode fazer o quê. `ADMIN` pode deletar qualquer coisa, `USER` só manipula seus próprios dados.

## @PreAuthorize

```java
@Service
@RequiredArgsConstructor
public class TarefaService {
    
    private final TarefaRepository repository;
    private final UserRepository userRepository;
    
    public List<Tarefa> listarTodas() {
        return repository.findAll();
    }
    
    @PreAuthorize("hasRole('ADMIN')")
    public void deletar(Long id) {
        repository.deleteById(id);
    }
    
    @PreAuthorize("#email == authentication.principal.username or hasRole('ADMIN')")
    public List<Tarefa> listarPorUsuario(String email) {
        return repository.findByUsuarioEmail(email);
    }
}
```

## Endpoint com Role

```java
@RestController
@RequestMapping("/api/admin")
@PreAuthorize("hasRole('ADMIN')")
public class AdminController {
    
    @GetMapping("/usuarios")
    public List<UserDTO> listarUsuarios() {
        return userRepository.findAll().stream()
            .map(UserDTO::new)
            .toList();
    }
    
    @DeleteMapping("/usuarios/{id}")
    public ResponseEntity<Void> deletarUsuario(@PathVariable Long id) {
        userRepository.deleteById(id);
        return ResponseEntity.noContent().build();
    }
}
```

## Roles Hierarchy

```java
@Bean
static RoleHierarchy roleHierarchy() {
    RoleHierarchyImpl hierarchy = new RoleHierarchyImpl();
    hierarchy.setHierarchy("ROLE_ADMIN > ROLE_USER");
    return hierarchy;
}
```
