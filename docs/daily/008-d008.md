## 🎯 O QUE VOCÊ VAI APRENDER HOJE

### Conceito:
**PUT, DELETE e Autenticação com curl**

### Definição em 1 frase:
PUT substitui um recurso inteiro, DELETE remove, e autenticação identifica quem está pedindo.

### Quando usar cada método:
```
GET    → Ler (não modifica nada)
POST   → Criar (gera novo ID)
PUT    → Substituir (substitui tudo)
PATCH  → Atualizar parcialmente (só muda o que precisa)
DELETE → Remover (apaga)
```

## 💻 CHECKLIST DE CÓDIGO

### 📁 Exercício 1: curl PUT (Substituir)

```bash
# PUT substitui o recurso inteiro
curl -X PUT https://jsonplaceholder.typicode.com/users/1 \
  -H "Content-Type: application/json" \
  -d '{
    "id": 1,
    "name": "Nome Atualizado COMPLETO",
    "username": "novoUsername",
    "email": "novo@email.com",
    "address": {
      "street": "Rua Nova",
      "city": "São Paulo"
    }
  }'
```

**⚠️ Importante:** PUT precisa enviar TODOS os campos!

**Checklist:**
- [ ] PUT usuário completo
- [ ] Verificar que todos campos foram substituídos
- [ ] Entender diferença PUT vs PATCH

---

### 📁 Exercício 2: curl PATCH (Atualizar parcialmente)

```bash
# PATCH atualiza só o que você enviar
curl -X PATCH https://jsonplaceholder.typicode.com/users/1 \
  -H "Content-Type: application/json" \
  -d '{
    "email": "email-atualizado@novo.com"
  }'
```

**⚠️ Importante:** PATCH só envia os campos que quer mudar!

**Checklist:**
- [ ] PATCH email
- [ ] Verificar que só email mudou
- [ ] Entender: PUT = tudo, PATCH = parcial

### 📁 Exercício 4: Autenticação

```bash
# Basic Auth (usuário:senha base64)
curl -u usuario:senha https://api.exemplo.com/perfil

# Bearer Token (JWT)
curl -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
     https://api.exemplo.com/dados-privados

# API Key (header customizado)
curl -H "X-API-Key: sua-chave-aqui" \
     https://api.exemplo.com/dados

# API Key (query param)
curl "https://api.exemplo.com/dados?api_key=sua-chave-aqui"
```

**Exemplo com JSONPlaceholder (não requer auth, mas simula):**
```bash
# Header Authorization (simula auth)
curl -H "Authorization: Bearer fake-token-123" \
     https://jsonplaceholder.typicode.com/users/1
```

**Checklist:**
- [ ] Basic Auth (-u)
- [ ] Bearer Token (-H)
- [ ] API Key header (-H)
- [ ] Entender quando usar cada um

### 📁 Exercício 5: API CRUD Completo

```bash
# 1. CREATE (POST)
echo "=== CREATE ==="
curl -s -X POST https://jsonplaceholder.typicode.com/users \
  -H "Content-Type: application/json" \
  -d '{"name": "Novo Usuario", "username": "novo", "email": "novo@email.com"}' | jq

# 2. READ (GET)
echo "=== READ ==="
curl -s https://jsonplaceholder.typicode.com/users/11 | jq '{name, email}'

# 3. UPDATE (PUT)
echo "=== UPDATE (PUT) ==="
curl -s -X PUT https://jsonplaceholder.typicode.com/users/11 \
  -H "Content-Type: application/json" \
  -d '{"name": "Usuario Modificado", "username": "modificado", "email": "mod@email.com"}' | jq '{name}'

# 4. PARTIAL UPDATE (PATCH)
echo "=== UPDATE (PATCH) ==="
curl -s -X PATCH https://jsonplaceholder.typicode.com/users/11 \
  -H "Content-Type: application/json" \
  -d '{"email": "novo-email@atualizado.com"}' | jq '{name, email}'

# 5. DELETE
echo "=== DELETE ==="
curl -s -X DELETE https://jsonplaceholder.typicode.com/users/11
echo ""
```

**Checklist:**
- [ ] CREATE → 201
- [ ] READ → 200
- [ ] UPDATE (PUT) → 200
- [ ] UPDATE (PATCH) → 200
- [ ] DELETE → 200
- [ ] Verificar depois do DELETE (404)
