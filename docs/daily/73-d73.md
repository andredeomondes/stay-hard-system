# Dia 73 — Spring Data JPA: @Query Customizada

## Conceito do Dia

Para queries complexas que o naming convention não suporta, use `@Query` com JPQL ou SQL nativo.

---

## Onde Estudar

- [Artigo: @Query](https://www.baeldung.com/spring-data-jpa-query)

---

## Checklist de Código

- [ ] Criar @Query com JPQL
- [ ] Criar @Query com SQL nativo

---

## JPQL (@Query)

```java
public interface TarefaRepository extends JpaRepository<Tarefa, Long> {
    
    @Query("SELECT t FROM Tarefa t WHERE t.categoria = :cat ORDER BY t.criadoEm DESC")
    List<Tarefa> buscarPorCategoriaOrdenado(@Param("cat") Categoria cat);
    
    @Query("SELECT t FROM Tarefa t WHERE t.concluida = false AND t.prioridade = :p")
    List<Tarefa> pendentesPorPrioridade(@Param("p") Prioridade p);
    
    @Query("SELECT COUNT(t) FROM Tarefa t WHERE t.categoria = :cat")
    long contarPorCategoria(@Param("cat") Categoria cat);
}
```

## SQL Nativo

```java
@Query(value = "SELECT * FROM tarefas WHERE DATE(criado_em) = CURRENT_DATE", 
       nativeQuery = true)
List<Tarefa> buscarDeHoje();
```

## Parâmetros

```java
// Posicional (não recomendado)
@Query("SELECT t FROM Tarefa t WHERE t.nome = ?1")
List<Tarefa> buscarPorNome(String nome);

// Nomeado (recomendado)
@Query("SELECT t FROM Tarefa t WHERE t.nome = :nome")
List<Tarefa> buscarPorNome(@Param("nome") String nome);
```

## Modifying Query (INSERT/UPDATE/DELETE)

```java
@Modifying
@Query("UPDATE Tarefa t SET t.concluida = true WHERE t.id = :id")
void marcarConcluida(@Param("id") Long id);
```

⚠️ **Não esqueça `@Modifying`** para UPDATE/DELETE!
