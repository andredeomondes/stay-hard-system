## 🎯 O QUE VOCÊ VAI APRENDER HOJE

### Conceito:
**Strategy — Trocar algoritmos em runtime**

### Definição em 1 frase:
Strategy permite trocar o comportamento de um objeto sem mudar sua classe.

### Quando usar:
- Quando precisa trocar algoritmos/estratégias
- Quando tem múltiplas variações de um comportamento
- Para evitar muitos if/else

## 💻 CHECKLIST DE CÓDIGO

### 📁 Exercício: Aplicar no LevelService

```java
// 1. Interface Strategy
interface LevelStrategy {
    String getLevelName(int daysCompleted);
}

// 2. Estratégias concretas
class NivelIniciante implements LevelStrategy {
    @Override
    public String getLevelName(int days) {
        if (days >= 75) return "Stay Hard";
        if (days >= 30) return "Unbreakable";
        return days >= 15 ? "Relentless" : days >= 7 ? "Forged" : "Awakening";
    }
}

class NivelIntermediario implements LevelStrategy {
    @Override
    public String getLevelName(int days) {
        return "Nível " + (days / 30 + 1);
    }
}

// 3. Context (usa a estratégia)
class LevelService {
    private LevelStrategy strategy;
    
    public LevelService(LevelStrategy strategy) {
        this.strategy = strategy;
    }
    
    public void setStrategy(LevelStrategy strategy) {
        this.strategy = strategy;
    }
    
    public String getLevelName(int days) {
        return strategy.getLevelName(days);
    }
}

// Uso
LevelService service = new LevelService(new NivelIniciante());
System.out.println(service.getLevelName(10)); // "Forged"

service.setStrategy(new NivelIntermediario());
System.out.println(service.getLevelName(10)); // "Nível 1"
```
