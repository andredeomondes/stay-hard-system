# Dia 49 — DDD: Entidades e Value Objects

## Conceito do Dia

**Entidade**: Tem identidade própria (ID). Dois objetos com mesmos atributos mas IDs diferentes são diferentes.
**Value Object**: Não tem identidade. É definido pelos seus atributos.

Exemplo: `CPF` é Value Object (123.456.789-00 é igual a 123.456.789-00). `Pessoa` é Entidade (mesmo nome, CNPJ diferente = pessoa diferente).

## Código: Entidade vs Value Object

```java
// VALUE OBJECT - imutável, sem identidade
@lombok.Value
public class Prioridade {
    public static final Prioridade BAIXA = new Prioridade("BAIXA", 1);
    public static final Prioridade MEDIA = new Prioridade("MEDIA", 2);
    public static final Prioridade ALTA = new Prioridade("ALTA", 3);
    
    String nome;
    int nivel;
}

// ENTIDADE - tem identidade
public class Tarefa implements Identidade<Tarefa> {
    private Long id;
    private String nome;
    private Prioridade prioridade; // Value Object
    private LocalDateTime criadoEm;
    
    @Override
    public Long getId() { return id; }
    
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Tarefa)) return false;
        Tarefa that = (Tarefa) o;
        return Objects.equals(id, that.id);
    }
    
    @Override
    public int hashCode() {
        return Objects.hash(id);
    }
}
```

## Aplicar no Stay Hard

```java
// Value Objects para Prioridade e Categoria
public enum Prioridade { BAIXA, MEDIA, ALTA }
public enum Categoria { ESTUDO, TRABALHO, SAUDE, LAZER }
```
