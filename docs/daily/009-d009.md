## 🎯 O QUE VOCÊ VAI APRENDER HOJE

### Conceito:
**HTTP Status Codes — Comunicação de resultados**

### Definição em 1 frase:
Status codes são números que o servidor retorna para dizer se a requisição deu certo ou não.

### Estrutura:
```
1xx → Informacional (processando)
2xx → Sucesso (deu certo)
3xx → Redirecionamento (vá para outro lugar)
4xx → Erro do Cliente (você fez algo errado)
5xx → Erro do Servidor (servidor fez algo errado)
```

## 💻 CHECKLIST DE CÓDIGO

### 📁 Exercício 1: Status 2xx (Sucesso)

```bash
# 200 OK — Requisição bem sucedida
curl -i https://jsonplaceholder.typicode.com/users/1
# Esperado: HTTP/2 200

# 201 Created — Recurso criado
curl -i -X POST https://jsonplaceholder.typicode.com/users \
  -H "Content-Type: application/json" \
  -d '{"name": "Test"}'
# Esperado: HTTP/2 201
# Header Location: /users/11

# 204 No Content — Sucesso sem body (comum em DELETE)
curl -i -X DELETE https://jsonplaceholder.typicode.com/users/1
# Esperado: HTTP/2 204
```

**Checklist:**
- [ ] 200 OK (GET, PUT)
- [ ] 201 Created (POST)
- [ ] 204 No Content (DELETE)
- [ ] Verificar headers com -i

---

### 📁 Exercício 2: Status 4xx (Erro do Cliente)

```bash
# 400 Bad Request — Requisição mal formada
curl -i -X POST https://jsonplaceholder.typicode.com/users \
  -H "Content-Type: application/json" \
  -d '{invalid json'
# Esperado: HTTP/2 400

# 401 Unauthorized — Não autenticado
curl -i https://private-api.example.com/users
# Esperado: HTTP/2 401

# 403 Forbidden — Autenticado mas sem permissão
curl -i -H "Authorization: Bearer token" \
     https://api.example.com/admin/users
# Esperado: HTTP/2 403

# 404 Not Found — Recurso não existe
curl -i https://jsonplaceholder.typicode.com/users/999
# Esperado: HTTP/2 404

# 422 Unprocessable Entity — Validação falhou
curl -i -X POST https://api.example.com/users \
  -H "Content-Type: application/json" \
  -d '{"email": "invalid-email"}'
# Esperado: HTTP/2 422

# 429 Too Many Requests — Rate limit
curl -i https://api.example.com/data
# Esperado: HTTP/2 429 (se muitas requisições)
```

**Checklist:**
- [ ] 400 Bad Request
- [ ] 401 Unauthorized
- [ ] 403 Forbidden
- [ ] 404 Not Found
- [ ] 422 Unprocessable
- [ ] Entender quando usar cada um

### 📁 Exercício 3: Status 5xx (Erro do Servidor)

```bash
# 500 Internal Server Error — Bug no servidor
# (simular com API que você controla)

# 502 Bad Gateway — Proxy/gateway error
curl -i https://502.example.com

# 503 Service Unavailable — Servidor fora
curl -i https://503.example.com

# 504 Gateway Timeout — Servidor demorou demais
curl -i --max-time 1 https://timeout.example.com
```

**⚠️ NUNCA exponha erros internos no frontend:**
```
❌ MAU: {"error": "java.lang.NullPointerException at UserService.java:45"}
✅ BOM: {"error": "Erro interno. Tente novamente."}
```

**Checklist:**
- [ ] Entender que 5xx = bug no servidor
- [ ] Entender que NUNCA mostrar stack trace
- [ ] Entender que 5xx = retry é ok

### 📁 Exercício 4: Status 3xx (Redirecionamento)

```bash
# 301 Moved Permanently — Redirecionamento permanente
# (Navegador cacheia)
curl -i https://httpbin.org/status/301

# 302 Found — Redirecionamento temporário
curl -i https://httpbin.org/status/302

# 304 Not Modified — Use cache (não baixa de novo)
curl -i -H "If-None-Match: abc123" https://example.com/data
```

**Checklist:**
- [ ] 301 vs 302
- [ ] 304 Not Modified
- [ ] Entender quando usar cada um

## 📊 RESUMO DOS STATUS CODES

### 1xx — Informacional
| Code | Quando usar |
|------|------------|
| 100 Continue | Pode continuar enviando |
| 101 Switching Protocols | Upgrade de protocolo |

### 2xx — Sucesso
| Code | Quando usar | Exemplo |
|------|-------------|---------|
| **200** | GET/PUT sucesso | `GET /users/1` |
| **201** | POST criou recurso | `POST /users` |
| **204** | DELETE sucesso, sem body | `DELETE /users/1` |

### 3xx — Redirecionamento
| Code | Quando usar |
|------|------------|
| 301 | Mover permanentemente |
| 302 | Mover temporariamente |
| 304 | Use cache |

### 4xx — Erro do Cliente
| Code | Quando usar | Solução |
|------|-------------|---------|
| **400** | Request mal formado | Corrigir request |
| **401** | Não autenticado | Fazer login |
| **403** | Sem permissão | Pedir acesso |
| **404** | Recurso não existe | Verificar URL |
| **409** | Conflito | Resolver conflito |
| **422** | Validação falhou | Corrigir dados |
| **429** | Rate limit | Esperar, retry |

### 5xx — Erro do Servidor
| Code | Quando usar | Solução |
|------|-------------|---------|
| 500 | Bug interno | Fix bug |
| 502 | Gateway error | Check proxy |
| 503 | Servidor fora | Tentar depois |
| 504 | Timeout | Tentar depois |
