# Dia 69 — Spring Boot: Profiles e Ambientes

## Conceito do Dia

Diferentes ambientes (dev, homolog, prod) têm configurações diferentes. Use **Spring Profiles** para separar configurações.

---

## Onde Estudar

- [Docs: Profiles](https://docs.spring.io/spring-boot/docs/current/reference/html/features.html#features.profiles)
- [Artigo: Multi-environment](https://www.baeldung.com/spring-profiles)

## Estrutura de arquivos

```
src/main/resources/
├── application.yml              # comum a todos
├── application-dev.yml          # só para dev
├── application-prod.yml         # só para prod
└── application-h2.yml           # banco em memória
```

## application-dev.yml

```yaml
spring:
  datasource:
    url: jdbc:h2:mem:testdb
  h2:
    console:
      enabled: true

logging:
  level:
    br.com.stayhard: DEBUG
```

## application-prod.yml

```yaml
spring:
  datasource:
    url: jdbc:postgresql://prod-server:5432/stayhard
    username: ${DB_USER}
    password: ${DB_PASSWORD}
  jpa:
    hibernate:
      ddl-auto: validate

logging:
  level:
    br.com.stayhard: INFO
    root: WARN
```

## @Profile em beans

```java
@Service
@Profile("dev")
public class DevEmailService implements EmailService {
    @Override
    public void enviar(String to, String msg) {
        log.info("[DEV] Email simulado para {}", to);
    }
}

@Service
@Profile("prod")
public class ProdEmailService implements EmailService {
    private final JavaMailSender mailSender;
    
    @Override
    public void enviar(String to, String msg) {
        SimpleMailMessage email = new SimpleMailMessage();
        email.setTo(to);
        email.setText(msg);
        mailSender.send(email);
    }
}
```

## Ativando profiles

```bash
# Por linha de comando
java -jar app.jar --spring.profiles.active=prod

# No application.yml
spring:
  profiles:
    active: dev
```
