# Dia 138 — Caching: Redis

## Conceito do Dia

**Redis** é um datastore em memória para caching, sessões, e pub/sub. Acelera leitura.

## Onde Estudar

## Docker Compose

```yaml
redis:
  image: redis:7-alpine
  ports:
    - "6379:6379"
  volumes:
    - redis_data:/data
```

## Dependência

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>
```

## Configuração

```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
      timeout: 2s
```

## Uso

```java
@Service
@RequiredArgsConstructor
public class TarefaService {
    
    private final TarefaRepository repository;
    private final RedisTemplate<String, Tarefa> redisTemplate;
    
    private static final String CACHE_KEY = "tarefa:";
    
    public Optional<Tarefa> buscarPorId(Long id) {
        String key = CACHE_KEY + id;
        
        // Tentar cache primeiro
        Tarefa cached = redisTemplate.opsForValue().get(key);
        if (cached != null) {
            log.debug("Cache hit for tarefa: {}", id);
            return Optional.of(cached);
        }
        
        // Cache miss - buscar do banco
        Optional<Tarefa> tarefa = repository.findById(id);
        
        tarefa.ifPresent(t -> {
            redisTemplate.opsForValue().set(key, t, Duration.ofMinutes(30));
        });
        
        return tarefa;
    }
    
    @CacheEvict(value = "tarefas", key = "#id")
    public Tarefa atualizar(Long id, AtualizarTarefaRequest request) {
        // ...
    }
}
```

## @Cacheable

```java
@Service
@CacheConfig(cacheNames = "tarefas")
public class TarefaService {
    
    @Cacheable(key = "#id")
    public Optional<Tarefa> buscarPorId(Long id) {
        return repository.findById(id);
    }
    
    @CacheEvict(allEntries = true)
    public Tarefa criar(Tarefa tarefa) {
        return repository.save(tarefa);
    }
}
```
