# Dia 81 — REST API: O que é REST?

## Conceito do Dia

**REST (Representational State Transfer)** é um estilo de arquitetura para APIs web. Usa HTTP verbs e URLs estruturadas.

## RESTful URLs

```
Método   URL                    Ação
-------  ---------------------  -----------------
GET      /api/tarefas           Listar todas
GET      /api/tarefas/1         Buscar por ID
POST     /api/tarefas           Criar
PUT      /api/tarefas/1         Atualizar (substituir)
PATCH    /api/tarefas/1         Atualizar (parcial)
DELETE   /api/tarefas/1         Deletar
```

## Resource Naming

```
# BOM (plural, nomes, não ações)
GET /api/tarefas
GET /api/tarefas/123
POST /api/tarefas

# RUIM (verbos, inconsistente)
GET /api/getTarefas
GET /api/tarefa/123
POST /api/criarTarefa
```

## REST != CRUD

Nem toda operação é CRUD. Use verbos quando necessário:

```
POST   /api/tarefas/123/concluir    → ação customizada
POST   /api/auth/login              → autenticação
GET    /api/tarefas/search?q=java   → busca
```
