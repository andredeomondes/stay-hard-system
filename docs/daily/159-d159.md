# Dia 159 — Security: HTTPS e SSL

## Conceito do Dia

**HTTPS** é obrigatório para produção. Configurar SSL/TLS corretamente.

---

## Onde Estudar

- [Let's Encrypt](https://letsencrypt.org/)

---

## Nginx com HTTPS

```nginx
server {
    listen 443 ssl http2;
    server_name stayhard.com;
    
    ssl_certificate /etc/letsencrypt/live/stayhard.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/stayhard.com/privkey.pem;
    
    # TLS 1.2+
    ssl_protocols TLSv1.2 TLSv1.3;
    
    # Ciphers seguros
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_prefer_server_ciphers off;
    
    # HSTS
    add_header Strict-Transport-Security "max-age=63072000" always;
    
    # OCSP Stapling
    ssl_stapling on;
    ssl_stapling_verify on;
}
```

## Spring Boot HTTPS

```yaml
server:
  ssl:
    enabled: true
    key-store: classpath:keystore.p12
    key-store-password: ${KEYSTORE_PASSWORD}
    key-store-type: PKCS12
    key-alias: stayhard
  port: 8443
```

## Security Headers

```nginx
add_header X-Frame-Options "SAMEORIGIN" always;
add_header X-Content-Type-Options "nosniff" always;
add_header X-XSS-Protection "1; mode=block" always;
add_header Referrer-Policy "strict-origin-when-cross-origin" always;
add_header Content-Security-Policy "default-src 'self'" always;
```

## Let's Encrypt com Certbot

```bash
# Instalar
apt install certbot python3-certbot-nginx

# Obter certificado
certbot --nginx -d stayhard.com -d www.stayhard.com

# Renew automático
certbot renew --dry-run
```
