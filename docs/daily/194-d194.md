# Dia 194 — Reactive: Spring WebFlux

## Conceito do Dia

**WebFlux** é Stack reativo não-bloqueante. Para alta concorrência com poucos recursos.

---

## Por que Reactive?

```
Traditional: 1 thread per request (bloqueante)
Reactive:    1 thread many requests (não-bloqueante)
```

## Dependência

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-webflux</artifactId>
</dependency>
```

## Flux e Mono

```kotlin
// Mono: 0 ou 1 elemento
fun buscarPorId(id: Long): Mono<Tarefa> {
    return repository.findById(id)
        .map { Optional.of(it) }
        .defaultIfEmpty(Optional.empty())
}

// Flux: 0 a N elementos
fun listarTodas(): Flux<Tarefa> {
    return repository.findAll()
}

fun listarPorCategoria(categoria: Categoria): Flux<Tarefa> {
    return repository.findByCategoria(categoria)
}
```

## Controller

```kotlin
@RestController
@RequestMapping("/api/tarefas")
class TarefaController(
    private val service: TarefaService
) {
    @GetMapping
    fun listar(): Flux<Tarefa> = service.listar()
    
    @GetMapping("/{id}")
    fun buscarPorId(@PathVariable id: Long): Mono<Tarefa> = 
        service.buscarPorId(id)
            .switchIfEmpty(Mono.error(RecursoNaoEncontradoException()))
    
    @PostMapping
    @ResponseStatus(CREATED)
    fun criar(@RequestBody @Valid request: CriarTarefaRequest): Mono<Tarefa> = 
        service.criar(request)
    
    @DeleteMapping("/{id}")
    @ResponseStatus(NO_CONTENT)
    fun deletar(@PathVariable id: Long): Mono<Void> = 
        service.deletar(id)
}
```

## Repository Reactive

```kotlin
@Repository
interface TarefaReactiveRepository : ReactiveCrudRepository<Tarefa, Long> {
    
    fun findByCategoria(categoria: Categoria): Flux<Tarefa>
    
    @Query("SELECT * FROM tarefas WHERE concluida = false")
    fun findPendentes(): Flux<Tarefa>
}
```
