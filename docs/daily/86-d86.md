# Dia 86 — REST API: CORS

## Conceito do Dia

**CORS (Cross-Origin Resource Sharing)** permite que seu front-end (outro domínio) consuma sua API.

---

## Onde Estudar

- [Artigo: CORS](https://www.baeldung.com/spring-cors)

---

## Config Global

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
            .allowedOrigins(
                "http://localhost:3000",      // React dev
                "https://meu-frontend.com"    // Produção
            )
            .allowedMethods("GET", "POST", "PUT", "PATCH", "DELETE", "OPTIONS")
            .allowedHeaders("*")
            .allowCredentials(true)
            .maxAge(3600);
    }
}
```

## Ou com @CrossOrigin (simples)

```java
@CrossOrigin(origins = "http://localhost:3000")
@RestController
@RequestMapping("/api/tarefas")
public class TarefaController {
    // ...
}
```

## Preflight Request

```
OPTIONS /api/tarefas HTTP/1.1
Origin: http://localhost:3000
Access-Control-Request-Method: POST

---
HTTP/1.1 204 No Content
Access-Control-Allow-Origin: http://localhost:3000
Access-Control-Allow-Methods: GET,POST,PUT,DELETE
```

---

## Erro comum

```
Access to fetch at 'http://localhost:8080/api/tarefas' from origin 
'http://localhost:3000' has been blocked by CORS policy
```

Solução: configurar CORS como mostrado acima.
