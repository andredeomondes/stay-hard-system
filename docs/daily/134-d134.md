# Dia 134 — Resilience: Circuit Breaker

## Conceito do Dia

**Circuit Breaker** previne falhas em cascata. Se um serviço falha, o CB "abre" e retorna fallback.

---

## Onde Estudar

- [Docs: Resilience4j](https://resilience4j.readme.io/)
- [Artigo: Circuit Breaker Pattern](https://martinfowler.com/bliki/CircuitBreaker.html)

---

## Dependência

```xml
<dependency>
    <groupId>io.github.resilience4j</groupId>
    <artifactId>resilience4j-spring-boot3</artifactId>
    <version>2.2.0</version>
</dependency>
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-circuitbreaker-resilience4j</artifactId>
</dependency>
```

## Configuração

```yaml
resilience4j:
  circuitbreaker:
    instances:
      tarefaService:
        slidingWindowSize: 10
        failureRateThreshold: 50
        waitDurationInOpenState: 10s
        permittedNumberOfCallsInHalfOpenState: 5
        automaticTransitionFromOpenToHalfOpenEnabled: true
```

## Uso

```java
@Service
@RequiredArgsConstructor
public class NotificacaoService {
    
    private final RestTemplate restTemplate;
    
    @CircuitBreaker(name = "tarefaService", fallbackMethod = "enviarFallback")
    public void enviarEmail(String to, String msg) {
        restTemplate.postForObject("http://email-service/api/email", 
            new EmailRequest(to, msg), Void.class);
    }
    
    private void enviarFallback(String to, String msg, Exception e) {
        // Salvar para retry posterior
        log.warn("Email falhou, salvo para retry: {} - {}", to, e.getMessage());
        salvarEmailPendente(to, msg);
    }
}
```

## Estados do Circuit Breaker

```
CLOSED (normal) ──→ OPEN (falhou)
    ↑                   │
    │                   ↓
    └── HALF_OPEN ◄─────┘
    (testando)
```
