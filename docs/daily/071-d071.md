# Dia 71 — Spring Data JPA: Introdução

## Conceito do Dia

**Spring Data JPA** simplifica persistência. Você escreve apenas a **interface** do Repository, e o Spring gera a implementação automaticamente!

"Não_WRITE implementation, only interface."

---

## Onde Estudar

- [Docs: Spring Data JPA](https://docs.spring.io/spring-data/jpa/docs/current/reference/html/)
- [Artigo: JPA vs JDBC](https://www.baeldung.com/jpa-vs-jdbc)

## Dependências

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>
```

## application.yml (dev)

```yaml
spring:
  datasource:
    url: jdbc:h2:mem:testdb
    driver-class-name: org.h2.Driver
    username: sa
    password: 
  h2:
    console:
      enabled: true
  jpa:
    hibernate:
      ddl-auto: create-drop
    show-sql: true
```

## Repository Mágico

```java
@Repository
public interface TarefaRepository extends JpaRepository<Tarefa, Long> {
    // TUDO que você precisa já está implementado!
}

// Métodos herdados:
// findById(Long) → Optional<Tarefa>
// findAll() → List<Tarefa>
// save(Tarefa) → Tarefa
// deleteById(Long) → void
// count() → long
// existsById(Long) → boolean
```
