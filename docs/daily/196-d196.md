# Dia 196 — SRE: Service Level Objectives

## Conceito do Dia

**SLOs** definem níveis de serviço aceitáveis. SRE garante que sejam cumpridos.

---

## Definição de SLOs

```
SLO: 99.9% uptime (3 nines)
     = 8.76 horas downtime/ano
     = 43 minutos downtime/mês

Error Budget: 0.1% = 100% - 99.9%
```

## SLIs (Service Level Indicators)

| SLI | Métrica |
|-----|---------|
| Disponibilidade | uptime / total_time |
| Latência | p95 response time |
| Throughput | requests / second |
| Error Rate | errors / total_requests |

## Error Budget Policy

```markdown
# Error Budget Policy

## SLOs
- Disponibilidade: 99.9%
- Latência p95: < 500ms
- Error Rate: < 0.1%

## Error Budget
- Mensal: 43.8 minutos de downtime permitido
- Diário: 1.44 minutos de downtime permitido

## Ação
- Budget < 50%: Alerta verde
- Budget < 25%: Alerta amarelo, pausar releases
- Budget < 0%: Incidente P1, freeze total
```

## Monitoring Config

```yaml
# alerts.yml
groups:
  - name: slo-alerts
    rules:
      - alert: SLOViolation
        expr: |
          (
            1 - (
              sum(rate(http_server_requests_seconds_count{status=~"5.."}[1h])) /
              sum(rate(http_server_requests_seconds_count[1h]))
            )
          ) * 100 > 0.1
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "SLO de disponibilidade violado"
          description: "Error rate > 0.1% por 5 minutos"

      - alert: LatencyHigh
        expr: |
          histogram_quantile(0.95, 
            rate(http_server_requests_seconds_bucket[5m])
          ) > 0.5
        for: 5m
        labels:
          severity: warning
```

## Runbook

```markdown
# Runbook: High Latency

## Diagnóstico
1. Verificar métricas de banco (p95 queries)
2. Verificar uso de CPU/memória
3. Verificar conexões pool

## Mitigação
1. Scale horizontal (mais instâncias)
2. Scale vertical (mais recursos)
3. Ativar circuit breaker
4. Rollback de deploy recente

## Comunicação
- Status page: updating
- Stakeholders: notify
- Post-mortem: required
```
