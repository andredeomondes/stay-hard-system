# Dia 41 — SOLID: Single Responsibility Principle

## Conceito do Dia

O **S** do SOLID: cada classe/função deve ter **uma única responsabilidade**. Não faça uma classe que calcula E salva E envia email.

## Mini-Projeto: Refatorar Stay Hard

```java
// ANTES: Classe Deus
class GerenciadorTarefa {
    void criarTarefa(String nome) { /* valida, salva, notifica */ }
    void listarTarefas() { /* busca, formata, exibe */ }
    void salvarEmArquivo() { /* serializa */ }
}

// DEPOIS: Responsabilidades separadas
class TarefaService { /* lógica de negócio */ }
class TarefaRepository { /* persistência */ }
class NotificacaoService { /* email/console */ }
```

## Aplicar no Stay Hard

Revise `TarefaService.java` e `Gerenciador.java` - extraia responsabilidades para classes específicas.
