# Dia 197 — Incident Management

## Conceito do Dia

Processos para responder e aprender de incidentes.

---

## Severity Levels

| Severity | Impact | Response Time | Example |
|----------|--------|---------------|---------|
| SEV1 | Total outage | 15 min | API down |
| SEV2 | Major degradation | 30 min | Slow responses |
| SEV3 | Minor issue | 2 hours | Non-critical bug |
| SEV4 | Low impact | Next sprint | UI glitch |

## Incident Lifecycle

```
DETECT → TRIAGE → RESPOND → RESOLVE → REVIEW
```

## Communication Template

```
## INCIDENT REPORT

**ID:** INC-2026-001
**Severity:** SEV2
**Status:** RESOLVED
**Started:** 2026-03-29 14:00 UTC
**Resolved:** 2026-03-29 14:45 UTC
**Duration:** 45 minutes

## Impact
- ~5000 users affected
- API latency increased to 10s
- Error rate: 15%

## Timeline
14:00 - Alert triggered: High latency
14:05 - On-call engaged
14:10 - Identified: Database connection pool exhausted
14:20 - Mitigation: Increased pool size
14:35 - Verified: Latency back to normal
14:45 - Declared resolved

## Root Cause
Connection pool max was 10, burst traffic caused exhaustion

## Action Items
- [ ] Increase default pool size to 50
- [ ] Add autoscaling based on connection count
- [ ] Add alert when pool > 80% utilization

## Lessons Learned
- Pool sizing was too conservative
- Need better capacity planning
```

## Post-mortem Template

```markdown
# Post-mortem: Database Connection Pool Exhaustion

## Summary
 spike in traffic caused connection pool exhaustion, leading to 45 minutes of degraded service.

## Root Cause
Connection pool configured with max 10 connections was insufficient during peak traffic.

## Contributing Factors
1. No autoscaling configured
2. Pool metrics not monitored
3. Capacity planning outdated

## Resolution
Increased max connections to 50 and added horizontal pod autoscaler.

## Action Items
| Item | Owner | Due Date | Status |
|------|-------|----------|--------|
| Add pool monitoring | DevOps | 2026-04-05 | Done |
| Update capacity plan | SRE | 2026-04-10 | In Progress |
| Implement HPA | DevOps | 2026-04-15 | Pending |
```
