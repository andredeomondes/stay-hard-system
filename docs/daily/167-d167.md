# Dia 167 — Agile: Scrum e Kanban

## Conceito do Dia

Metodologias ágeis organizam o desenvolvimento em ciclos curtos e entregas incrementais.

---

## Onde Estudar

- [Scrum Guide](https://www.scrumguides.org/)
- [Kanban](https://kanban.university/)

---

## Scrum

| Evento | Frequência | Duração |
|---------|------------|---------|
| Sprint | 1-4 semanas | Fixada |
| Daily | Diário | 15 min |
| Sprint Planning | Início | 2-4h |
| Sprint Review | Final | 1-2h |
| Sprint Retrospective | Final | 1-2h |

## Papéis

| Papel | Responsabilidade |
|-------|------------------|
| Product Owner | Priorização, ROI |
| Scrum Master | Facilitar, remover obstáculos |
| Dev Team | Entregar incremento |

## Kanban (mais simples)

```
| TO DO | IN PROGRESS | REVIEW | DONE |
|--------|-------------|--------|-------|
| Task 5 | Task 2     | Task 1 |      |
| Task 6 |            |        |      |
```

## Quadro Stay Hard

```markdown
## Sprint 4: MVP Completo

### TO DO
- [ ] Implementar search
- [ ] Adicionar filtros
- [ ] Dark mode

### IN PROGRESS
- [ ] Autenticação JWT

### REVIEW
- [ ] CRUD tarefas

### DONE
- [x] Setup projeto
- [x] Docker config
```
