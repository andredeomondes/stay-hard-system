# Dia 156 — DevOps: Ansible

## Conceito do Dia

**Ansible** automatiza configuração de servidores e deploy de aplicações.

---

## Onde Estudar

- [Docs: Ansible](https://docs.ansible.com/ansible/latest/getting_started/)

---

## Instalação

```bash
pip install ansible
# ou
brew install ansible
```

## Inventário

```ini
# inventory.ini
[webservers]
web1.stayhard.com ansible_host=10.0.1.10
web2.stayhard.com ansible_host=10.0.1.11

[database]
db1.stayhard.com ansible_host=10.0.2.10

[all:vars]
ansible_user=deploy
ansible_ssh_private_key_file=~/.ssh/id_rsa
```

## Playbook

```yaml
# deploy.yml
---
- hosts: webservers
  become: yes
  vars:
    app_dir: /opt/stayhard
    app_version: "{{ version }}"

  tasks:
    - name: Instalar dependências
      apt:
        name:
          - nginx
          - certbot
        state: present

    - name: Criar diretório da aplicação
      file:
        path: "{{ app_dir }}"
        state: directory
        owner: deploy
        group: deploy

    - name: Deployar nova versão
      get_url:
        url: "https://artifacts.stayhard.com/{{ app_version }}/app.jar"
        dest: "{{ app_dir }}/app.jar"
        owner: deploy
        group: deploy

    - name: Reiniciar aplicação
      systemd:
        name: stayhard
        state: restarted
        daemon_reload: yes

    - name: Configurar Nginx
      template:
        src: nginx.conf.j2
        dest: /etc/nginx/sites-available/stayhard
      notify: Reload nginx

  handlers:
    - name: Reload nginx
      service:
        name: nginx
        state: reloaded
```

## Executar

```bash
ansible-playbook -i inventory.ini deploy.yml --extra-vars "version=1.2.3"
```
